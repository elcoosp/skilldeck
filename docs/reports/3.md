## Issue 1 — Lint warnings ignore global/individual rule disable (skill detail panel re-lint)

**Root cause:** `useDisableRule` in `src/hooks/use-skills.ts` only invalidates `['lint-rules']`, but the skill detail panel's lint re-run calls `commands.lintSkill` (or equivalent) which reads `AppState.lint_config` via `Arc<RwLock<LintConfig>>`. The problem is **two-fold**:

1. On re-lint from the detail panel, the skill scanner is called with a stale snapshot of `LintConfig` that was cloned at startup — it doesn't read the current `Arc<RwLock>`.
2. The individual-card "disable this rule" action likely calls `disableLintRule` with `scope: 'workspace'` but the in-memory `lint_config` on `AppState` is only updated for global scope.

**Fix:**

`src-tauri/src/commands/skills.rs` — in the `lint_skill` (or `relint_skill`) command, acquire a fresh read-lock on `state.lint_config` and pass it into the scanner:
```rust
let config = state.lint_config.read().await.clone();
let warnings = scanner::scan_skill(&skill_path, &config).await?;
```

In `src-tauri/src/commands/settings.rs` → `disable_lint_rule`, after updating the persisted TOML, also update the in-memory arc so all subsequent lints see the new state immediately:
```rust
let mut config = state.lint_config.write().await;
config.disabled_rules.push(rule_id.clone()); // or set severity to off
```

`src/hooks/use-skills.ts` — after `useDisableRule` succeeds, also invalidate `['local_skills']` so the card rerenders with fresh warnings:
```ts
onSuccess: () => {
  queryClient.invalidateQueries({ queryKey: ['lint-rules'] })
  queryClient.invalidateQueries({ queryKey: ['local_skills'] })
}
```

## Issue 2 — Code block horizontal scrollbar not using `thin-scrollbar`, text wrapping

**Root cause:** `App.css` applies `.thin-scrollbar` to the vertical axis but the code block renderer (likely via `rehype` plugins + `prose` class) sets `overflow-x: auto` without the thin-scrollbar variant, and code blocks need `white-space: pre-wrap` to avoid horizontal overflow on long lines.

**Fix in `src/App.css`:**
```css
/* target the scroll container rehype emits for code blocks */
.prose pre,
pre[class*="language-"] {
  overflow-x: auto;
  overflow-y: hidden;
  white-space: pre-wrap;       /* wrap long lines instead of scrolling */
  word-break: break-all;
  scrollbar-width: thin;       /* Firefox */
  scrollbar-color: var(--scrollbar-thumb) transparent;
}

.prose pre::-webkit-scrollbar,
pre[class*="language-"]::-webkit-scrollbar {
  height: 4px;                 /* horizontal track */
}
.prose pre::-webkit-scrollbar-thumb,
pre[class*="language-"]::-webkit-scrollbar-thumb {
  background: var(--scrollbar-thumb);
  border-radius: 2px;
}
```

If you want hard wrapping on content that should scroll (like minified JSON), remove `white-space: pre-wrap` and keep only the scrollbar fix.

## Issue 4 — Bookmarks don't survive restart (rely only on DB)

**Root cause:** `src/store/bookmarks.ts` is a Zustand store that hydrates from `localStorage` (or in-memory). On restart the Tauri webview resets, losing the in-memory state. The DB has a `bookmarks` table and there are commands in `src-tauri/src/commands/bookmarks.rs`, but the store doesn't reconcile with the DB on mount.

**Fix in `src/store/bookmarks.ts`:** Remove any `persist` middleware backed by `localStorage`. Instead, make bookmarks purely server-driven:

```ts
// Remove zustand persist, make the store just a cache for the DB result
export const useBookmarkStore = create<BookmarkStore>((set) => ({
  bookmarks: [],
  setBookmarks: (bookmarks) => set({ bookmarks }),
}))
```

Add a `useBookmarks` React Query hook that calls `commands.listBookmarks()` on mount with `staleTime: 0`. Wherever bookmarks are displayed, use this hook rather than the Zustand store. The store can remain as a local cache that is always seeded from the DB query result.

**Root cause:** `src/components/conversation/message-input.tsx` uses a `<textarea>` with `rows` auto-resize but without a `ScrollArea`. When content overflows, native scrollbar appears without the thin-scrollbar treatment and there's no max-height cap with overflow scroll.

**Fix in `message-input.tsx`:**

```tsx
import { ScrollArea } from '@/components/ui/scroll-area'

// Replace the raw <textarea> wrapper with:
<ScrollArea
  className={cn(
    'transition-all duration-150 ease-out',
    contentRows > 3 ? 'max-h-48' : 'max-h-none'
  )}
  style={{ height: `${clampedHeight}px` }}
>
  <textarea
    ref={textareaRef}
    className="w-full resize-none bg-transparent outline-none thin-scrollbar p-0"
    style={{ height: '100%', overflow: 'hidden' }}
    ...
  />
</ScrollArea>
```

Compute `clampedHeight` from the textarea's `scrollHeight` capped at a max (e.g. `Math.min(scrollHeight, 192)`). Apply a Framer Motion `animate={{ height: clampedHeight }}` wrapper for the smooth growth animation.
