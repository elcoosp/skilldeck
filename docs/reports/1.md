Below is the analysis of the user pain points followed by actionable design solutions.

### 🔍 Usability Analysis & Heuristic Evaluation

**1. The "Context Dissonance" Bug (File Picker & Skill Discovery)**
*   **Heuristic Violation:** *Consistency and Standards* & *Match between System and Real World*.
*   **User Impact:** Critical. Users operate within a mental model of "Active Workspace." When the file picker or skill detector looks at the binary launch directory instead of the DB-stored workspace, it breaks the user's trust in the application's awareness of their environment. It feels like the app is "blind" to the user's current location.
*   **Behavioral Insight:** Users expect spatial consistency. If I open a project, *everything* should happen relative to that project.

**2. The "Phantom Conversation" (New chats missing from panel)**
*   **Heuristic Violation:** *Visibility of System Status*.
*   **User Impact:** High. When a user creates a new chat, they seek immediate visual confirmation (feedback). If the chat doesn't appear, users assume the action failed and may retry, creating duplicate empty chats, or feel anxious about whether their data is safe.
*   **Root Cause Hypothesis:** A state synchronization disconnect between the backend event (Chat Created) and the frontend UI state (List Update).

**3. The "Blocked Flow" (Input disabled during Agent run)**
*   **Heuristic Violation:** *User Control and Freedom*.
*   **User Impact:** Medium-High. This disrupts the "stream of consciousness" workflow. Users often formulate their next thought while the AI is processing the previous one. Forcing them to wait creates "dead time" and increases perceived latency.
*   **User Need:** The ability to "draft" or "queue" thoughts without interrupting the current process.

**4. The "Timeless History" (Date/recency bugged)**
*   **Heuristic Violation:** *Recognition rather than Recall*.
*   **User Impact:** Medium. Users rely on temporal landmarks to navigate history ("I want that chat from yesterday"). If everything is "Recently," the list becomes an undifferentiated stream, forcing the user to scan every title, increasing cognitive load.

### 💡 Recommended UX Solutions

Here are the proposed solutions, formatted as design specifications to guide the development team.

#### 1. Contextual Root Binding
**name**: Contextual Root Binding
**description**: Refactors all file system operations (file picker, skill discovery) to resolve paths relative to the `active_workspace` path stored in the database, rather than the process's Current Working Directory (CWD). This ensures the application "lives" where the user is working, not where the binary is launched.
**color**: #4A90E2 (Stability Blue)
**emoji**: ⚓
**vibe**: "Grounded and reliable; the app finally knows where it stands."

#### 2. Instant Visibility Sync
**name**: Instant Visibility Sync
**description**: Implements a reactive state update mechanism (e.g., optimistic UI updates or WebSocket events) to immediately render new conversation headers in the left panel upon creation. This closes the feedback loop, confirming to the user that their action succeeded before the backend even finishes persisting.
**color**: #50E3C2 (Feedback Teal)
**emoji**: ⚡
**vibe**: "Responsive and alive; every action gets an immediate reaction."

#### 3. Non-Blocking Draft Mode
**name**: Non-Blocking Draft Mode
**description**: Transforms the input field from a "disabled" state to a "draft" state during agent execution. The input field remains editable, allowing users to type or paste context. A "Queue" button appears instead of "Send", allowing users to submit the text immediately once the agent finishes, or a "Cancel Draft" option to discard.
**color**: #F5A623 (Interaction Orange)
**emoji**: ⌨️
**vibe**: "Fluid and respectful; the app waits for the user, not the other way around."

#### 4. Precise Temporal Display
**name**: Precise Temporal Display
**description**: Fixes the date comparison logic (likely a timezone offset or formatting string error) to accurately render relative timestamps (e.g., "Yesterday", "2 hours ago", "March 14"). Ensures the `created_at` timestamp is correctly retrieved from the database and converted to the user's local time zone.
**color**: #7ED321 (Clarity Green)
**emoji**: 📅
**vibe**: "Clear and organized; history makes sense again."
