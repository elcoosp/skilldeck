### Issue 1 — Cancel button has no effect; no UI reflection; missing retry button

**Root cause:** The Stop button in `message-input.tsx` calls `commands.cancelAgent` and sets `agentRunning` to false, but nothing in the message thread or message bubble reflects the cancelled state. `MessageData` has no `is_cancelled` flag surfaced, and there is no retry affordance on user messages.

**Plan:**
- In `src-tauri/skilldeck-models/src/messages.rs`, add a `status` field (or reuse an existing one) that the agent loop sets to `"cancelled"` when `cancelAgent` is invoked, then emits an `AgentEvent::Cancelled` — the existing event type already exists.
- In `src-tauri/skilldeck-core/src/agent/loop.rs`, on receipt of the cancellation signal, mark the last assistant message `status = "cancelled"` in the DB before returning.
- In `src/hooks/use-agent-stream.ts`, listen to the `cancelled` AgentEvent and invalidate the messages query so the thread re-fetches.
- In the `MessageBubble` component (`src/components/conversation/message-bubble.tsx`), detect `message.role === 'assistant' && message.status === 'cancelled'` and show a subtle "Cancelled" badge on the message.
- On the preceding user message, when the next assistant message is `cancelled`, show a **Retry** button. Clicking it re-submits the same content via `useSendMessage`. The simplest approach: pass `onRetry?: () => void` as a prop from `MessageThread`, which calls `sendMutation.mutateAsync({ content: message.content })`.

### Issue 3 — Paste long text then select-all-delete doesn't readjust height; line-by-line delete does

**Root cause:** The height recalculation effect in `message-input.tsx` sets `el.style.height = 'auto'` then reads `scrollHeight`, which works when content shrinks character-by-character. However after a bulk delete, React batches the state update and the DOM hasn't reflowed before `scrollHeight` is read, so the height stays stale.

**Plan:**
- Replace the current imperative `useEffect` with a `useLayoutEffect` (which runs synchronously after DOM mutations):
```ts
useLayoutEffect(() => {
  const el = textareaRef.current
  if (!el) return
  el.style.height = '0px'      // force reflow before reading scrollHeight
  el.style.height = `${Math.min(el.scrollHeight, MAX_HEIGHT)}px`
}, [content])
```
Setting to `'0px'` first (rather than `'auto'`) forces a synchronous reflow so `scrollHeight` reflects the actual required height with the new (possibly empty) content.
- Also drop the `motion.div` animated height wrapper or at minimum keep `contentHeight` in sync via the same `useLayoutEffect`.

  3. Checks the code fence `meta` string (the text after the language identifier on the opening ` ``` ` line) for a filename pattern.
  4. Stores resolved `data-language` and `data-filename` attributes on the `<pre>` element.

  - Otherwise: show the language string (e.g. `typescript`, `python`).
  - If neither is found: show nothing (hide the label area rather than showing `"code"`).
