## Tier 1 — Critical Bugs (ship first)

### 1.1 🔴 Auto-name conversation from first message

**Files:** `src/hooks/use-conversations.ts`, `src/hooks/use-agent-stream.ts`  
**Status:** Not firing at all.

**Implementation** (use the `persisted` event placement — cleanest, no race condition):

```typescript
// src/hooks/use-agent-stream.ts — inside processEvent, case 'persisted':
case 'persisted':
  queryClient.invalidateQueries({ queryKey: ['messages', conversationId] })

  const conversations = queryClient.getQueryData<any[]>(['conversations'])
  const currentConvo = conversations?.find(c => c.id === conversationId)
  if (currentConvo && !currentConvo.title) {
    const messages = queryClient.getQueryData<any[]>(['messages', conversationId])
    const firstUserMsg = messages?.find(m => m.role === 'user')
    if (firstUserMsg?.content) {
      const raw = firstUserMsg.content.trim().slice(0, 60)
      const title = raw.charAt(0).toUpperCase() + raw.slice(1)
      commands.renameConversation(conversationId, title)
        .then(() => queryClient.invalidateQueries({ queryKey: ['conversations'] }))
    }
  }

  queryClient.invalidateQueries({ queryKey: ['conversations'] })
  break
```

Expose a `useAutoNameConversation` mutation in `use-conversations.ts` so the same logic can be called from the message input as a fallback if the stream path is unavailable.

**Acceptance criteria:** After first message completes streaming, sidebar conversation title updates to the first 60 chars of the message. Subsequent messages do not rename.

### 1.7 🔴 Make all assistant message code blocks clickable (URLs)

**Files:** `src/components/conversation/message-bubble.tsx` (or the Markdown renderer component).

Two sub-tasks:

**A — Detect and linkify URLs in code blocks**

The markdown renderer likely uses `rehype-highlight` or similar. Add a post-processing step or a custom rehype plugin that wraps bare URLs inside `<code>` and `<pre>` elements with `<a>` tags:

```typescript
// Custom rehype plugin skeleton
import { visit } from 'unist-util-visit'
import { isElement } from 'hast-util-is-element'

export function rehypeLinkifyCodeUrls() {
  return (tree: Root) => {
    visit(tree, 'element', (node) => {
      if (!isElement(node, 'code')) return
      // walk text nodes, find URLs, wrap in <a href="...">
    })
  }
}
```

**B — Open links correctly** (depends on fix 1.3)

All `<a>` tags inside the renderer must use Tauri's opener, not `href` navigation:

```tsx
// In your markdown component's link override:
components={{
  a({ href, children }) {
    return (
      <a
        href={href}
        onClick={(e) => {
          e.preventDefault()
          if (href) open(href) // Tauri opener
        }}
      >
        {children}
      </a>
    )
  }
}}
```

**Acceptance criteria:** URLs inside code blocks are visually underlined/styled as links; clicking them opens the correct URL in the system browser.
