We believe that great documentation is never finished – it evolves with user needs. That's why we have a structured process for reviewing feedback every week. This page explains how we do it and how you can see the impact of your contributions.

## Why We Do This

- **User‑driven improvements**: Feedback tells us what's working and what's confusing.
- **Rapid iteration**: Weekly reviews mean issues are addressed quickly.
- **Transparency**: You can see what we're working on and what's been fixed.

## The Process

### 1. Collect Feedback

Feedback comes from multiple sources:

- **"Was this helpful?" buttons** on every page (with optional comments)
- **GitHub issues** tagged `documentation`
- **Discord** messages in the `#docs-feedback` channel
- **User interviews** and usability tests (quarterly)

All feedback is aggregated into a private dashboard (internal use only).

### 2. Weekly Review Meeting

Every **Monday at 15:00 UTC**, the docs team (and any interested contributors) meets to review feedback from the previous week. The agenda:

- **New feedback** – what did users say?
- **Top themes** – are there recurring issues?
- **Priority items** – which problems should we tackle first?
- **Assignments** – who will work on what?

Meeting notes are published in the [GitHub Discussions](https://github.com/elcoosp/skilldeck/discussions/categories/docs-feedback) for transparency.

### 3. Triage and Prioritization

We use a simple priority system:

| Priority | Criteria | Target Resolution |
|----------|----------|-------------------|
| **P0 (Critical)** | Broken links, factual errors, security issues | Within 24 hours |
| **P1 (High)** | Confusing explanations, missing key content | Within 1 week |
| **P2 (Medium)** | Typos, style improvements, additional examples | Within 2 weeks |
| **P3 (Low)** | Nice‑to‑have enhancements, future ideas | Backlog |

### 4. Take Action

Depending on the issue, we may:

- Fix it immediately (small changes)
- Create a GitHub issue for larger tasks
- Add it to the documentation roadmap
- Respond to the user with an update

### 5. Close the Loop

When a feedback item is resolved, we:

- Update the original issue/comment with a link to the fix
- If the user provided contact info, we notify them directly
- Add a note to the next "You Spoke, We Listened" update

## Monthly "You Spoke, We Listened" Updates

At the end of each month, we publish a summary of the feedback we received and the changes we made. This helps the community see the impact of their contributions. [Read the latest update →](/en/community/you-spoke-we-listened)

## How You Can Participate

- **Submit feedback** using the buttons on any page.
- **Join the weekly review** – all are welcome! Contact us on Discord for the meeting link.
- **Tag issues** with `documentation` to bring them to our attention.
- **Contribute fixes** – if you see a problem, you can submit a PR directly.
