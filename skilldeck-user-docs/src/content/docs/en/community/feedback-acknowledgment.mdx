When you take the time to give us feedback, you deserve to know that we've heard you – and what we're doing about it. This page explains our feedback acknowledgment system and how we keep you informed.

## How It Works

### 1. You Submit Feedback

You can submit feedback in several ways:

- Click the **"Was this page helpful?"** buttons at the bottom of any documentation page (optional comment).
- Open a **GitHub issue** with the `documentation` label.
- Share feedback in the **#docs-feedback** channel on Discord.
- Participate in **user interviews** or **usability tests** (occasional).

### 2. We Acknowledge Receipt

- **Immediate acknowledgment**: When you submit feedback via the docs site, you'll see a confirmation message and receive a reference ID (e.g., `FEED-2026-1234`). You can use this ID to track progress.
- **GitHub issues**: You'll get an automated comment confirming receipt and a label assigned.
- **Discord**: A bot or human will react to your message within 24 hours.

### 3. We Triage and Act

Your feedback enters our weekly review process (see [Weekly Feedback Review](/en/community/feedback-review-process)). We categorize it, prioritize it, and assign it to a team member.

### 4. We Update You

When there's progress, we notify you:

- **If you provided contact info** (email or Discord), we send a direct message.
- **GitHub issues** are updated with comments.
- **Monthly "You Spoke, We Listened" updates** summarize all changes (see [latest update](/en/community/you-spoke-we-listened)).

### 5. We Close the Loop

When your feedback is resolved (e.g., a documentation change is merged), we:

- Post a final update with a link to the change.
- Mark the GitHub issue as closed.
- Send a final notification (if contact info provided).

## What to Expect

| Feedback Type | Acknowledgment | Resolution Update |
|--------------|----------------|-------------------|
| Bug report (typo, broken link) | Immediate (automated) | When fix is merged |
| Content suggestion | Within 1 week (after review) | When decided (implemented or deferred) |
| Feature request | Within 2 weeks (after triage) | When roadmap updated |
| General comment | Acknowledged, no further update | N/A |

## Tracking Your Feedback

If you have a reference ID, you can check the status by:

- Visiting the [Feedback Dashboard](https://feedback.skilldeck.dev) (internal tool, not public yet)
- Replying to the acknowledgment email or Discord message
- Mentioning the ID in a new GitHub issue

We're working on a public dashboard where you'll be able to see all feedback items and their status.

## Why We Do This

We believe that feedback is a gift. By acknowledging and acting on it, we:

- Build trust with our community
- Continuously improve the documentation
- Show that every voice matters

Thank you for helping us make SkillDeck better!
