Welcome, and thank you for your interest in contributing to the SkillDeck documentation! This guide will walk you through setting up your local environment, making changes, and submitting your first pull request.

## Prerequisites

Before you start, ensure you have the following installed:

- **Git** – [Download Git](https://git-scm.com/downloads)
- **Node.js** (v20 or later) – [Download Node.js](https://nodejs.org/) or use [nvm](https://github.com/nvm-sh/nvm)
- **pnpm** (package manager) – Install with `npm install -g pnpm` or [other methods](https://pnpm.io/installation)
- A **code editor** – We recommend [VS Code](https://code.visualstudio.com/)

## Step 1: Fork and Clone the Repository

1. Go to the [SkillDeck repository](https://github.com/elcoosp/skilldeck) and click the **Fork** button (top‑right).
2. Clone your fork locally:
   ```bash
   git clone https://github.com/your-username/skilldeck.git
   cd skilldeck
   ```
3. Add the original repository as an upstream remote to keep your fork synced:
   ```bash
   git remote add upstream https://github.com/elcoosp/skilldeck.git
   ```

## Step 2: Set Up the Docs Site

The documentation site lives in the `skilldeck-user-docs/` folder.

1. Navigate to the docs folder:
   ```bash
   cd skilldeck-user-docs
   ```
2. Install dependencies:
   ```bash
   pnpm install
   ```
3. Start the development server:
   ```bash
   pnpm run dev
   ```
4. Open `http://localhost:4321` in your browser. The site will automatically reload when you save changes.

## Step 3: Understand the Folder Structure

All content is in `src/content/docs/`. The structure is:

```
src/content/docs/
  en/
    latest/          # Latest version (active development)
      getting-started/
      how-to/
      explanation/
      reference/
      community/
      market-insights/
    v0-2/            # Previous versions (mostly auto‑generated)
    v0-1/
  fr/                # French translations (similar structure)
```

Most contributions should go under `en/latest/`. When adding a new page, include frontmatter at the top with `title` and `description` fields.

## Step 4: Make Your Changes

- **Edit existing pages** – simply open the relevant `.md` or `.mdx` file and make your changes.
- **Add a new page** – create a new `.md` file in the appropriate directory. Use the frontmatter template:

  ```yaml
  ---
  title: "Your Page Title"
  description: "A brief description of the page."
  order: 5  # optional, for sidebar ordering
  ---
  ```

- **Use our custom components** – you can import and use `Nudge`, `Checkpoint`, and `Feedback` in any `.mdx` file (they are already globally available). Example:

  ```mdx
  import { Nudge } from '../../../components/mdx/Nudge.astro';

  <Nudge type="tip" title="Pro tip">
    This is a helpful tip!
  </Nudge>
  ```

## Step 5: Preview Your Changes

While the dev server is running, you can see your changes live at `http://localhost:4321`. Check that everything looks correct, especially:

- Spelling and grammar
- Links work
- Code blocks are properly formatted
- No unintended layout changes

## Step 6: Commit and Push

1. Create a new branch for your changes:
   ```bash
   git checkout -b docs/your-change-description
   ```
2. Commit your changes with a clear message:
   ```bash
   git add .
   git commit -m "docs: fix typo in installation guide"
   ```
3. Push your branch to your fork:
   ```bash
   git push origin docs/your-change-description
   ```

## Step 7: Open a Pull Request

1. Go to your fork on GitHub and click **Compare & pull request**.
2. Ensure the base repository is `elcoosp/skilldeck` and the base branch is `main`.
3. Give your PR a clear title and description, explaining what you changed and why.
4. If your PR fixes an issue, include `Fixes #123` in the description.
5. Click **Create pull request**.

A maintainer will review your changes. If any changes are requested, simply push new commits to the same branch – the PR will update automatically.

## Tips for Success

- **Keep changes focused** – one PR should address one issue or topic.
- **Test your changes** – run the dev server and check for errors.
- **Follow our style guide** – use second person, active voice, and present tense. See the [style guide](https://github.com/elcoosp/skilldeck/blob/main/STYLEGUIDE.md) for details.
- **Be patient** – reviews can take a few days, especially for larger contributions.

## Getting Help

If you get stuck, you can:

- Ask in the `#docs` channel on [Discord](https://discord.gg/skilldeck)
- Start a [GitHub Discussion](https://github.com/elcoosp/skilldeck/discussions)
- Open a draft PR and ask for feedback early

Thank you for contributing to SkillDeck – your efforts help make the project better for everyone!
