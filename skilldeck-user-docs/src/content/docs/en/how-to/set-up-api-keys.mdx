To use cloud‑based AI providers like **Claude (Anthropic)** or **OpenAI**, you need to add an API key. This key is stored securely in your operating system’s keychain and never touches the SkillDeck database. If you prefer to stay entirely local, you can use **Ollama** without any API key.

<Nudge type="tip" title="What you'll learn">
  How to obtain and securely store API keys for Claude and OpenAI, and how to verify they work.
</Nudge>

## Prerequisites

- SkillDeck installed and running
- An account with the provider you want to use (Anthropic or OpenAI)

## Step 1: Open Settings

1. Click the **Settings** icon (gear) in the left sidebar.
2. Navigate to the **API Keys** tab.

## Step 2: Obtain Your API Key

### For Claude (Anthropic)

1. Go to the [Anthropic Console](https://console.anthropic.com/).
2. Sign in and navigate to **API Keys**.
3. Click **Create Key**, give it a name (e.g., "SkillDeck"), and copy the generated key.

<Nudge type="warning" title="Key starts with sk-ant">
  Claude API keys always begin with `sk-ant-`. If yours doesn't, double‑check you're in the right place.
</Nudge>

### For OpenAI

1. Go to the [OpenAI Platform](https://platform.openai.com/).
2. Navigate to **API Keys** under your profile.
3. Click **Create new secret key**, copy it immediately.

<Nudge type="warning" title="Key starts with sk-">
  OpenAI keys typically start with `sk-`. Treat this key like a password – never share it or commit it to version control.
</Nudge>

### For Ollama (No Key Needed)

If you're using Ollama, you don't need an API key. Just ensure Ollama is installed and running locally. SkillDeck will automatically detect it.

<Checkpoint question="Do you have your API key copied?" onYes="Great! Let's add it to SkillDeck." onNo="Pause here and generate a key from the provider's console." />

## Step 3: Add the Key in SkillDeck

1. In the **API Keys** tab, find the provider you're setting up (Claude or OpenAI).
2. Click the **Add** or **Edit** button next to it.
3. Paste your key into the input field. You can toggle visibility to double‑check it.
4. Click **Save**.

The key is now stored in your system keychain. You can verify it's saved by looking for a "Key stored" badge next to the provider.

<Nudge type="tip" title="Key storage">
  SkillDeck uses your OS keychain (macOS Keychain, Windows Credential Manager, or libsecret on Linux). Keys are never written to the SQLite database.
</Nudge>

## Step 4: Test the Connection (Optional but Recommended)

After saving, you can test that the key works:

1. Go to the **Profiles** tab and create a profile using that provider (if you haven't already).
2. Start a new conversation with that profile.
3. Send a simple message like "Hello, are you working?".

If you get a response, your key is correctly configured.

<Checkpoint question="Did you receive a response from the model?" onYes="Perfect! Your API key is working." onNo="Check that you selected the correct profile and that the key was pasted correctly." />

## Step 5: Manage Multiple Keys

You can only have one active key per provider at a time. If you need to switch to a different key (e.g., a team key vs. personal key), simply replace the existing one by pasting a new key and saving. The old key will be overwritten.

## Troubleshooting

- **Invalid key error** – double‑check that you copied the entire key, including any prefixes.
- **Key not saved** – ensure your system's keychain is unlocked and accessible.
- **Provider not appearing** – only Claude, OpenAI, and Ollama are supported out‑of‑the‑box. Other providers may be added in the future.
