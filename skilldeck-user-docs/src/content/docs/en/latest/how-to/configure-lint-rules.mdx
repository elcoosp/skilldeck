SkillDeck includes a powerful linter that checks your skills for common issues, security risks, and quality problems. You can customize which rules are active and their severity using configuration files.

<Nudge type="tip" title="What you'll learn">
  How to create and edit `skilldeck-lint.toml` to tailor linting to your needs.
</Nudge>

## Understanding Lint Rules

Lint rules are grouped into categories:

- **Frontmatter** – validates skill metadata (name, description, license, etc.)
- **Structure** – checks file existence, size, and references
- **Security** – detects dangerous patterns (e.g., shell commands, obfuscated code)
- **Quality** – encourages clear instructions, examples, and freshness

Each rule has a severity: `off`, `info`, `warning`, or `error`. By default, most rules are set to `warning` or `error` for security‑critical ones.

## Configuration File Locations

Lint settings can be defined in two places (workspace settings override global ones):

- **Global**: `~/.config/skilldeck/skilldeck-lint.toml` – applies to all skills on your machine
- **Workspace**: `./.skilldeck/skilldeck-lint.toml` – applies only to skills in the current workspace

<Nudge type="tip" title="Creating the file">
  If the file doesn't exist, you can create it manually. Use the default configuration as a starting point.
</Nudge>

## Default Configuration

The default configuration looks like this:

```toml
[defaults]
severity = "warning"

[rules]
# Override severity for specific rules:
# "fm-name-format" = "error"
# "sec-dangerous-tools" = "error"
# "fm-license-present" = "off"

[rule_params]
# Rule-specific parameters can go here
```

## Common Customizations

### Disable a Rule Entirely

To turn off a rule (e.g., if you find it too strict), add it to the `[rules]` section with value `"off"`:

```toml
[rules]
"fm-description-length" = "off"
```

### Change Rule Severity

You can make a rule more or less severe. For example, to treat missing license as an error (instead of info):

```toml
[rules]
"fm-license-present" = "error"
```

### Configure Rule Parameters

Some rules accept parameters. For example, you can adjust the maximum allowed skill size (default 100 KB):

```toml
[rule_params]
"struct-skill-md-size" = { max_bytes = 51200 }  # 50 KB
```

## Example: Strict Security Settings

If you're reviewing community skills and want maximum safety:

```toml
[defaults]
severity = "error"

[rules]
# Keep security rules as errors, make quality warnings also errors
"quality-content-examples" = "error"
"quality-content-steps" = "error"
```

## Example: Relaxed Rules for Personal Skills

For your own skills, you might want fewer warnings:

```toml
[rules]
"fm-description-length" = "off"
"quality-content-examples" = "off"
"struct-references-depth" = "info"
```

## Applying Changes

After editing the configuration file:

1. Save the file.
2. Re‑run the linter on your skill:
   ```bash
   skilldeck-lint ~/.agents/skills/my-skill
   ```
3. The new severity levels will be reflected in the output.

<Checkpoint question="Did the linter output change according to your new config?" onYes="Great! Your custom rules are working." onNo="Check the TOML syntax and file location." />

## Viewing Available Rules

To see all rules and their default severity, run:

```bash
skilldeck-lint --list-rules
```

This will output a list of rule IDs with descriptions.
