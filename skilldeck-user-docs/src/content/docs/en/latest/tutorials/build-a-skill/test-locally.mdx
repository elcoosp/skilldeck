You've written your skill's frontmatter and instructions. Now it's time to see it in action! This tutorial will guide you through testing your skill in a live SkillDeck conversation.

<Nudge type="tip" title="Prerequisites">
  - SkillDeck installed and running
  - Your skill directory placed in `~/.agents/skills/` (personal location)
  - A conversation open in SkillDeck
</Nudge>

## Step 1: Ensure Your Skill is Loaded

SkillDeck automatically scans your personal skill directory (`~/.agents/skills/`) on startup. If you added the skill while SkillDeck was already running, you may need to restart the app or trigger a rescan:

- **Restart SkillDeck** – the easiest way to ensure your skill is picked up.
- Alternatively, use the **Skills tab** in the right panel: click the refresh icon (circular arrow) to force a rescan.

<Checkpoint question="Does your skill appear in the Skills tab (under 'Local' or 'Installed')?" onYes="Great! It's loaded and ready." onNo="Check that the skill directory is exactly in `~/.agents/skills/` and that the folder name matches the skill `name` in frontmatter." />

## Step 2: Open a Conversation

Navigate to the **Conversation** tab (first icon in the right sidebar). If you don't have an active conversation, create a new one by clicking **New Chat** in the left panel.

## Step 3: Invoke Your Skill

In the message input, type `@` followed by your skill name. For example, if your skill is named `my-awesome-skill`, type:

```
@my-awesome-skill
```

As you type, a command palette should appear showing matching skills. Select your skill from the list. You'll see it appear as a chip above the input.

<Checkpoint question="Does your skill appear in the command palette?" onYes="Perfect! Now let's give it a task." onNo="Make sure you typed the exact skill name. If it doesn't appear, try restarting SkillDeck again." />

## Step 4: Give Your Skill a Task

After mentioning the skill, type a message that asks it to perform its intended function. For example, if your skill reviews Python code:

```
Can you review this function?

def add(a, b):
    return a + b
```

Press Enter to send the message.

## Step 5: Observe the Response

The agent will process your request, using your skill's instructions. Watch the response – it should follow the guidelines you provided in your skill.

If the response isn't what you expected, don't worry – you'll refine it in the next step.

<Checkpoint question="Did the response use your skill's instructions?" onYes="Awesome! Your skill works." onNo="Let's debug." />

## Step 6: Debugging and Refining

If the skill didn't behave as expected, consider these common issues:

- **Instructions too vague** – add more concrete examples.
- **Missing context** – if your skill needs file access, ensure the required MCP servers are running.
- **Name mismatch** – the skill name in frontmatter must match the directory name.
- **Linting warnings** – run `skilldeck-lint .` in your skill directory to catch errors.

After making changes to your `SKILL.md`, save the file. SkillDeck's hot‑reload should pick up changes automatically. If not, restart SkillDeck.

## Step 7: Iterate

Repeat steps 3‑6 until you're happy with the results. Testing and refining is a normal part of skill development.
