SkillDeck's frontend communicates with the Rust backend via **Tauri commands**. These commands are auto‑generated into TypeScript bindings, providing type‑safe access to all backend functionality.

## Command Categories

Commands are grouped by domain. Below is a high‑level overview; for the complete list with exact signatures, see the [auto‑generated bindings](#full-reference).

### Conversations
- `create_conversation` – Start a new conversation for a profile.
- `list_conversations` – Return recent conversations (optionally filtered by profile).
- `delete_conversation` – Soft‑delete a conversation.
- `rename_conversation` – Change a conversation's title.
- `pin_conversation` / `unpin_conversation` – Pin/unpin for quick access.

### Messages
- `list_messages` – Retrieve messages for a conversation (optionally for a branch).
- `send_message` – Send a user message and trigger the agent loop.
- `resolve_tool_approval` – Approve or deny a pending tool call.

### Skills
- `list_skills` – Get all loaded skills (local and registry).
- `toggle_skill` – Enable or disable a skill.
- `install_skill` – Copy a skill from the registry to local storage.
- `uninstall_skill` – Remove an installed skill.
- `diff_skill_versions` – Compare local skill with registry version.
- `lint_skill` – Run the linter on a skill directory.

### MCP Servers
- `list_mcp_servers` – List all configured MCP servers.
- `add_mcp_server` – Add a new server (stdio or SSE).
- `remove_mcp_server` – Delete a server configuration.
- `connect_mcp_server` / `disconnect_mcp_server` – Manually control connection state.

### Profiles & API Keys
- `list_profiles` – Get all profiles.
- `create_profile` / `update_profile` / `delete_profile` – Manage profiles.
- `set_default_profile` – Choose the default profile.
- `list_api_keys` – Check which providers have stored keys.
- `set_api_key` / `delete_api_key` – Manage API keys (stored in OS keychain).

### Workspaces
- `list_workspaces` – Get all known workspaces.
- `open_workspace` – Open a folder as a workspace (detects project type).
- `close_workspace` – Mark a workspace as closed.

### File System (for context injection)
- `list_directory_contents` – List files in a directory (for the `#` picker).
- `count_folder_files` – Count files shallow/deep for folder attachment.
- `read_file` – Read a file's content (with size limit).
- `assemble_folder` – Concatenate multiple files for context.

### Queued Messages
- `add_queued_message` – Add a message to the queue.
- `list_queued_messages` – Get queued messages for a conversation.
- `update_queued_message` – Edit a queued message.
- `delete_queued_message` – Remove from queue.
- `reorder_queued_messages` – Change order.
- `merge_queued_messages` – Combine multiple queued messages.

### Platform Features
- `ensure_platform_registration` – Register with SkillDeck Platform.
- `get_platform_preferences` – Fetch user preferences (email, nudges, etc.).
- `update_platform_preferences` – Update preferences.
- `get_pending_nudges` – Retrieve unread nudges.
- `send_activity_event` – Send analytics event (if opted in).
- `get_referral_stats` – View referral progress.
- `create_referral_code` – Generate a referral link.

### Utilities
- `list_ollama_models` – Get locally installed Ollama models.
- `cancel_agent` – Stop the currently running agent loop.
- `open_folder` – Reveal a folder in the system file explorer.

## Full Reference

The complete, up‑to‑date command signatures are maintained in:

**`src/lib/bindings.ts`** (in the frontend source)

This file is automatically generated by [Tauri Specta](https://github.com/oscartbeaumont/tauri-specta) and includes:

- All command names and their parameter/return types.
- Type definitions for all request/response objects (e.g., `ConversationSummary`, `McpServerResponse`).
- Helper functions for invoking commands with type safety.

### Example Usage

```typescript
import { commands } from '@/lib/bindings';

// List all conversations
const result = await commands.listConversations(null, null);
if (result.status === 'ok') {
  console.log(result.data); // typed as ConversationSummary[]
} else {
  console.error(result.error);
}
```
