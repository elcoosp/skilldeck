The evaluator‑optimizer pattern is perfect for tasks that benefit from iterative refinement – like writing, code generation, or translation. A generator step produces an initial output, an evaluator step checks it, and if it's not good enough, the generator tries again, optionally with feedback.

<Nudge type="tip" title="What you'll build">
  A workflow that writes a short story, evaluates it, and revises it until it meets quality criteria.
</Nudge>

## Prerequisites

- SkillDeck installed and running
- Familiarity with [sequential](/en/tutorials/create-a-workflow/sequential) workflows

## Step 1: Open the Workflow Editor

1. Go to the **Workflow** tab.
2. Click **New Workflow**.

## Step 2: Understand the Evaluator‑Optimizer Pattern

The workflow runs in a loop until the evaluator passes:

1. **Generator** – creates an output based on a prompt.
2. **Evaluator** – assesses the output and returns `PASS` or `FAIL` (and optionally feedback).
3. If the evaluator fails, the generator runs again, possibly incorporating feedback.

The loop stops after a maximum number of attempts (default 5) or when the evaluator passes.

## Step 3: Create the Workflow Definition

Copy the following JSON into the editor:

```json
{
  "name": "Story Writer with Refinement",
  "pattern": "evaluator_optimizer",
  "steps": [
    {
      "id": "generator",
      "name": "Write a short story",
      "prompt": "Write a very short story (about 100 words) on the theme: {{theme}}"
    },
    {
      "id": "evaluator",
      "name": "Evaluate the story",
      "prompt": "Evaluate the following story. If it's engaging, well‑structured, and grammatically correct, respond with only the word PASS. Otherwise, respond with FAIL and a brief explanation of what to improve.\n\n{{generator.output}}"
    }
  ],
  "dependencies": [
    { "from": "generator", "to": "evaluator" }
  ]
}
```

<Checkpoint question="Did you paste the JSON correctly?" onYes="Great! Now save it." onNo="Check for syntax errors." />

## Step 4: Save the Workflow

1. Click **Save** and name it "Story Writer".
2. It should appear in your saved workflows.

## Step 5: Run the Workflow

1. Click **Run**.
2. When prompted for `theme`, enter something like `"a robot learning to paint"`.
3. Click **Start** and watch the process.

## Step 6: Observe the Iteration

The workflow will:

- Generate a story.
- Evaluate it.
- If the evaluator fails, the generator runs again (you might see multiple cycles).
- Eventually, either a PASS occurs or the max attempts are reached.

<Checkpoint question="Did the workflow produce a story (possibly after a few tries)?" onYes="Excellent! You've built an evaluator‑optimizer workflow." onNo="Check the evaluator prompt – ensure it expects 'PASS' or 'FAIL' exactly. Also make sure your model provider supports the required format." />

## Customizing the Evaluator

You can make the evaluator stricter or more lenient by adjusting its prompt. For example, you could ask for a numeric score and then treat above 7 as PASS.
