SkillDeck can be configured via a TOML file. This reference covers all available options, their default values, and examples.

## Configuration File Location

The configuration file is named `config.toml` and is stored in the platform‑specific config directory:

| OS      | Path                                                                 |
|---------|----------------------------------------------------------------------|
| macOS   | `~/Library/Application Support/skilldeck/config.toml`               |
| Linux   | `~/.config/skilldeck/config.toml`                                   |
| Windows | `%APPDATA%\skilldeck\config.toml` (e.g., `C:\Users\YourName\AppData\Roaming\skilldeck\config.toml`) |

If the file does not exist, SkillDeck uses built‑in defaults. You can create it manually to override settings.

## File Format

The configuration uses [TOML](https://toml.io/) (v1.0.0). It is divided into sections:

```toml
[platform]
# Platform‑related settings

[agent]
# Agent loop settings
```

## Platform Settings

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `enabled` | boolean | `true` | Whether to enable connection to SkillDeck Platform (community skills, nudges, referrals). |
| `url` | string | `"https://platform.skilldeck.dev"` | Base URL of the platform API. Change only if you are using a self‑hosted instance. |

**Example:**
```toml
[platform]
enabled = true
url = "https://platform.skilldeck.dev"
```

## Agent Settings

| Key | Type | Default | Description |
|-----|------|---------|-------------|
| `max_eval_opt_iterations` | integer | `5` | Maximum number of generator‑evaluator cycles in an evaluator‑optimizer workflow before giving up. |

**Example:**
```toml
[agent]
max_eval_opt_iterations = 10
```

## Full Example

```toml
[platform]
enabled = true
url = "https://platform.skilldeck.dev"

[agent]
max_eval_opt_iterations = 5
```

## Notes

- Changes to the configuration file require a restart of SkillDeck to take effect.
- Invalid TOML syntax or unknown keys will be ignored (with a warning in the logs).
- If you are running SkillDeck from source, the config directory is relative to the user's home as shown above.
