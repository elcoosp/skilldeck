SkillDeck connects to external tools and data sources via **MCP (Model Context Protocol)** servers. MCP is an open standard maintained by Anthropic and the community.

## Official MCP Documentation

For a complete introduction to MCP, server configuration, and available transports (stdio, SSE), please visit:

🔗 **[modelcontextprotocol.io/docs/getting-started/intro](https://modelcontextprotocol.io/docs/getting-started/intro)**

The official docs cover:

- What MCP is and why it matters
- How to set up an MCP server
- The JSON‑RPC message format
- Available transports and their configuration

## How SkillDeck Uses MCP

SkillDeck acts as an **MCP client**. When you add a server in the MCP tab, SkillDeck:

- Spawns the server process (for stdio) or connects via SSE.
- Performs the initialization handshake.
- Discovers available tools.
- Routes tool calls from the agent to the appropriate server.

All server‑specific configuration (command, arguments, environment variables) is handled through the SkillDeck UI. For transport‑level details, refer to the official MCP docs.
