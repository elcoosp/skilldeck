Have you built a skill that others might find useful? Great! The SkillDeck community thrives on shared knowledge and reusable components. Here's how you can contribute your skill.

## Two Ways to Share

### 1. Publish as a GitHub Gist (Simple)

The easiest way to share a skill is to publish it as a [GitHub Gist](https://gist.github.com). This creates a public, shareable URL that anyone can use to import your skill directly into SkillDeck.

See the [Share via Gist tutorial](/en/tutorials/build-a-skill/share-via-gist) for step‑by‑step instructions.

### 2. Get Featured in the Showcase

If you'd like your skill to appear on the [Skill Showcase](/en/community/skill-showcase) page, follow these steps:

1. Ensure your skill is published as a Gist (or hosted in a public GitHub repository).
2. Open an issue in the [SkillDeck GitHub repository](https://github.com/elcoosp/skilldeck/issues) with the title "Skill Showcase Submission".
3. Include the following information:
   - **Skill name**
   - **Your name / GitHub handle**
   - **Short description** (1‑2 sentences)
   - **Link** to the Gist or repository
   - **Screenshot / demo** (optional)
4. A maintainer will review your submission and add it to the showcase if it meets our quality guidelines.

## Quality Guidelines

To be featured in the showcase, your skill should:

- Have a clear, descriptive name (kebab‑case).
- Include a complete `SKILL.md` with frontmatter and instructions.
- Be tested and work as expected.
- Not contain malicious or dangerous code.
- Include a license (preferably open source).

## Share Your Success

We love hearing how you're using SkillDeck! Even if you don't want to submit to the showcase, feel free to share your creations in the `#skills` channel on [Discord](https://discord.gg/skilldeck). You might inspire others!
