Les espaces de travail sont un moyen de donner à l'agent accès aux fichiers et à la structure de votre projet. Lorsque vous ouvrez un dossier en tant qu'espace de travail, SkillDeck le scanne et rend son contenu disponible pour l'injection de contexte.

## Pourquoi utiliser des espaces de travail ?

- **Conscience du projet** : l'agent peut référencer les fichiers de votre projet sans copier‑coller manuel.
- **Aide contextuelle** : posez des questions sur votre codebase et obtenez des réponses basées sur les fichiers réels.
- **Développement de compétences** : testez les compétences par rapport à du code réel.
